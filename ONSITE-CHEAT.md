## 0. 단 하나만 기억해라
**노트북은 그냥 브라우저다.** 작업은 전부 클라우드(클라우드셸)에서 돈다. 윈도우든 뭐든 상관없음.
대회는 **노코드** — 샘플 코드를 클라우드셸에 받아서, 하네스로 프롬프트 붙여넣고 모델 고르고 배포 명령 몇 줄 돌리는 흐름. `[확정]`

## 1. 접속 (제일 먼저)
1. 데스크탑 브라우저 바로가기 실행 → **catalog.workshops.aws/join**

3. **리전 = us-east-1** (어제 ap-northeast-1과 다름 — 모델 액세스도 여기서 확인) `[확정]`

## 2. 작업 경로 선택 = 클라우드셸 + 하네스
워크숍이 3가지 경로(Local Machine / Harness / CloudShell)를 주는데 → **클라우드셸 + 하네스**로 가라(환경문제 회피, 안내지 권장). `[확정]`

## 3. 4단계 흐름 (안내지 그대로)
1. **워크숍 내용 훑기** — 규칙·구조 빠르게. "1단계: 팀 구축 – 샘플 코드 받기"까지 진행.
2. **클라우드셸 열기** — 콘솔에서 CloudShell 검색 → "단계1: 저장소 클론"의 4줄 명령 실행 → 샘플 코드 클론.
3. **전술 도구 배포** — "3단계: 에이전트 강화 – AgentCore Gateway"로 가서 **"5개 Agent 배포 7(5+2)" 명령** 실행 → 람다 + 게이트웨이 자동 생성.
4. **하네스로 Agent 5개 생성** — 프롬프트 붙여넣기 + 모델 선택 + agent 생성 + 게이트웨이. ★여기서 우리 키트가 들어감.

## 4. 우리 키트(shfoot) 쓰는 법 — "배포"가 아니라 "브레인"
대회가 노코드라 우리 squad.py를 직접 배포하진 않음. 대신:
1. **프롬프트**: prompts/gk.md·def.md·mid.md·fwd.md → 하네스 Agent 만들 때 **그대로 붙여넣기.** 전원 공 쫓기 금지(anti-swarm) 역할 규율이 승부처.
2. **정책 숫자**: policy.py 상단 존 좌표·슛레인지·압박트리거 → 하네스 설정/프롬프트에 반영.
3. **스키마 검증**: 클라우드셸은 리눅스 → 거기서
   ```
   git clone https://github.com/nexusailabs/shfoot && cd shfoot
   python3 reconcile.py            # 내장 샘플로 GREEN 확인
   ```
   샘플 관측(obs) 한 개 잡아서 `python3 reconcile.py obs.json` → 키 이름/좌표계 검증.
